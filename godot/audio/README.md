# Аудиосистема Ellipsis

Аудиофайлы намеренно не назначены. Откройте ресурс нужного события в Godot,
разверните `Layers` и назначьте файл в поле `Stream` каждого слоя.

## События

| Ресурс | Событие |
| --- | --- |
| `events/resonator_place_player.tres` | установка резонатора игроком |
| `events/resonator_place_enemy.tres` | установка резонатора противником |
| `events/wave_launch_purple.tres` | запуск фиолетовой волны |
| `events/wave_launch_blue.tres` | запуск синей волны |
| `events/wave_launch_red.tres` | запуск красной волны |
| `events/crossbar_install_quick.tres` | быстрая установка ковырялки |
| `events/crossbar_install_charged.tres` | заряженная установка ковырялки |

Один `SoundLayer` — один одновременно проигрываемый компонент звука. Поэтому
удар, механический щелчок и хвост реверберации можно хранить отдельными слоями
и настраивать независимо. Варианты одного и того же слоя удобно назначать через
встроенный `AudioStreamRandomizer` в поле `Stream`.

У слоя доступны громкость, случайный разброс громкости и высоты, задержка,
вероятность проигрывания, шина и параметры затухания 2D-звука. У события —
ограничение одновременных голосов, cooldown и приоритет.

## Музыка

Музыкальные точки запуска находятся в `audio/music/`:

- `dialogue.tres` — обычные диалоги, старт с 27,0 с;
- `rooms.tres` — обычные комнаты, старт с 0,0 с;
- `rahn_dialogue.tres` — диалог с Рахном, старт с 7,4 с;
- `rahn_battle.tres` — бой с Рахном, старт с 56,0 с;
- `demo_clear.tres` — завершение демо, `The Clockwork Garden.mp3` со 176,0 с.

Каждая точка зацикливается от своей стартовой позиции до конца композиции.
Громкость всех музыкальных точек — 56% от исходной (`-5,036 dB`).
Нарастание и перекрёстный переход между музыкальными темами занимают 0,4 с.
После смерти Рахна боевая музыка отдельно затихает за 1,25 с; финальная тема
включается только при достижении выхода и появлении надписи о прохождении демо.

Базовая громкость шины `Music` — 90% (`-0,915 dB`). Шина фонем `Dialogue`
работает на 80% (`-1,938 dB`) и дополнительно наследует общий уровень `SFX`.

## Вызовы из кода

```gdscript
Audio.play_2d(&"wave.launch.red", global_position)
Audio.play_global(&"ui.confirm")
Audio.play_music(&"dialogue")
Audio.play_music(&"rahn_battle")
Audio.set_bus_volume_linear(&"SFX", 0.8)
```

Новые события добавляются в `catalogs/main_sfx_catalog.tres`, а музыкальные
композиции — в `catalogs/main_music_catalog.tres`.

## Фонемы диалогов

Десять событий `dialogue.voice.*` находятся в `audio/events/`, а их WAV-файлы —
в `assets/audio/dialogue/`. Все они направлены в шину `Dialogue`, которая входит
в `SFX` и поэтому подчиняется общей настройке громкости эффектов.

`InterludeOverlay.gd` выводит реплики посимвольно и вызывает фонему при появлении
каждых 2–3 значимых символов. Пробелы не озвучиваются, а после знаков препинания
делается короткая пауза. Первое нажатие ЛКМ, Enter или Space раскрывает текущую
строку целиком; второе переходит к следующей.

Общая скорость появления реплик и пауз увеличена в 1,7 раза константой
`TYPEWRITER_SPEED_MULTIPLIER`.

Голос можно назначить сообщению явно:

```gdscript
{
	"voice": "violet",
	"characters_per_second": 38.0,
	"characters_per_sound": 2,
	"speaker": {"en": "VIOLET", "ru": "ФИОЛЕТОВЫЙ"},
	"text": {"en": "...", "ru": "..."},
}
```

Без явного поля `voice` система автоматически распознаёт текущих говорящих
`COLORLESS`, `CRON` и `RAHN`. Доступные профили: `colorless`, `cron`, `rahn`,
`violet`, `irvel`, `tiu`, `golden`, `orum`, `varn`, `hollow_armor`.
